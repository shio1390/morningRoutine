# あさじたく

子どもが朝の支度を自分で進めるための、完全ローカルPWAです。サーバーや外部ライブラリは使わず、ルーチン、順番、完了状態、よく使う項目、通知設定をすべてブラウザのIndexedDBに保存します。

## 開発方法

ES ModulesとService Workerを使うため、`index.html`をダブルクリックして`file://`で直接開くことはできません。ローカルHTTPサーバーで起動してください。

Windowsでは、プロジェクトフォルダで次を実行するのが最も簡単です。

```powershell
.\start-local.ps1
```

その後、ブラウザで [http://localhost:8080](http://localhost:8080) を開きます。終了はターミナルで`Ctrl+C`です。

手動で起動する場合は以下でも動作します。

```powershell
python -m http.server 8080
```

`file:///.../index.html` は、CORSにより`app.js`と`manifest.json`の読み込みに失敗するため使用しないでください。データを初期化するときは、ブラウザの開発者ツールからサイトのIndexedDBを削除してください。

## Netlifyへのデプロイ

1. このフォルダをGitHub等へpushするか、NetlifyのDeploys画面へフォルダをドラッグ＆ドロップします。
2. Build commandは不要です。Publish directoryはプロジェクトのルート（`.`）にします。
3. 発行されたHTTPS URLを開くとPWAとService Workerが有効になります。

ビルド不要の静的サイトなので、Netlify Functionsや環境変数の設定は不要です。

## PWAのインストール

- Android/Chrome: ブラウザメニューから「アプリをインストール」または「ホーム画面に追加」。
- iPhone/iPad/Safari: 共有ボタンから「ホーム画面に追加」。
- PC/Chrome: アドレスバー右側のインストールアイコンを選択。

インストールとオフライン利用にはHTTPS（Netlify）または`localhost`が必要です。

## 通知の制限

通知をオンにするとブラウザの許可を求めます。Web標準だけでは、アプリ・ブラウザが完全に閉じている間に指定時刻で確実に通知を発火させる仕組みはありません。このアプリは開いている間に1分ごとに朝・前日の設定時刻を確認して通知します。確実なバックグラウンド通知が必要な場合は、将来Push APIと通知サーバーを追加してください（現仕様ではサーバーを使用しません）。

## データについて

設定画面からJSON形式でエクスポート・インポートできます。ブラウザのサイトデータを消去するとIndexedDBの内容も消えるため、定期的なエクスポートをおすすめします。
